## 3. Limpeza do Histórico Git

### Problema

Um arquivo de backup (`backups/atlas_pre_frontend_20260509_090317.zip`) de **3.55 GB** havia sido commitado no início do histórico do repositório. Mesmo com `backups/` adicionado ao `.gitignore` posteriormente, o objeto permanecia no pack do git, ocupando **3.30 GiB** no `.git/`.

### Solução

```bash
brew install git-filter-repo
git filter-repo --path backups/atlas_pre_frontend_20260509_090317.zip --invert-paths --force
git gc --aggressive --prune=now
```

### Resultado

| Métrica | Antes | Depois |
|---------|-------|--------|
| `.git/` | 3.3 GB | 8.3 MB |
| Size-pack | 3.30 GiB | 8.09 MiB |
| Objetos | 1.346 | 1.317 |
| Commits | 46 | 38 (8 commits vazios removidos) |

O histórico foi reescrito — todos os commit hashes mudaram. O remote (`origin`) foi restaurado e o push forçado já foi executado.

## 5. Recuperação

Todo código removido está preservado em `mef_engine/_archive/`. Para restaurar um arquivo específico:

```bash
git mv mef_engine/_archive/<arquivo>.py mef_engine/
```

Para restaurar tudo:

```bash
mv mef_engine/_archive/*.py mef_engine/
```

Para rebuildar o ambiente de desenvolvimento:

```bash
make build-rust        # Compila o Rust + instala wheel
cd mef_frontend && npm run dev   # Frontend principal
cd mef_dashboard && npm run dev  # Dashboard
```
